You are a world-class Landing Page Strategist who has worked at Unbounce, studied 10,000+ high-converting pages, and helped scale 500+ SaaS companies past $1M ARR.

I need a complete, conversion-optimized landing page for a SaaS Studio.

Context:
- Product: Custom full-stack development and high-converting SaaS landing pages built with modern tech and AI integrations
- Target audience: SaaS founders, startups, and indie builders who need fast, scalable products but lack reliable developers who can ship high-quality apps and landing pages quickly
- Price point: High ticket
- Traffic source: Organic + Social
- Primary goal: Demos / discovery calls

Now build the entire landing page, section by section:

1. HERO SECTION
- Headline (max 8 words — attack the #1 pain point)
- Subheadline (max 20 words — show the transformation)
- Primary CTA button text (max 4 words, action-oriented)
- Secondary CTA (low-commitment alternative)
- Hero visual description (what image/animation to use)
- Trust bar (5 logos or "Trusted by X+ companies")

2. PROBLEM AGITATION
- 3 pain points your audience feels daily
- Use "you" language — make it visceral
- End with a transition line that introduces the solution

3. SOLUTION SHOWCASE
- 3 core features, each with:
  → Feature name (3 words max)
  → Benefit-driven description (15 words max)
  → Icon/visual suggestion
- Frame features as the bridge from pain → desired outcome

4. SOCIAL PROOF ENGINE
- 3 testimonials (write realistic placeholders I can swap later):
  → Quote (2 sentences max)
  → Name, role, company
  → Specific metric or result
- Stats bar: 3 credibility numbers (users, results, time saved)

5. HOW IT WORKS
- 3 dead-simple steps (verb + outcome format)
- Make the process feel effortless
- Include a visual journey description

6. OBJECTION CRUSHER
- 5 FAQ questions that secretly handle objections:
  → "Is this right for me?" → qualification
  → "What if it doesn't work?" → risk reversal
  → "Why is this different?" → differentiation
  → "How long until I see results?" → timeline
  → "What's the catch?" → transparency

7. FINAL CTA SECTION
- Urgency-driven headline
- Before → After transformation recap
- Primary CTA (same as hero for consistency)
- Risk reversal statement (guarantee / free trial)
- Scarcity element if applicable

8. FRAMER BUILD SPECS
- Color palette (5 hex codes — primary, secondary, accent, dark, light)
- Font pairing (Google Fonts only)
- Section spacing rhythm (consistent padding)
- Mobile-first responsive breakpoints
- Scroll-triggered animation suggestions
- Page speed optimization notes

Format everything so I can paste each section directly into Framer as individual components.
